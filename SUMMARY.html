System Architecture & Design Summary

1. Architecture Overview

The receiver is architected around a jitter buffer utilizing a std::map structure to manage packet sequencing. Incoming packets are buffered to mitigate network jitter, while the system continuously monitors sequence numbers to identify gaps. Upon detecting a missing sequence number, the receiver proactively generates NACKs (Negative Acknowledgments) to request retransmission, ensuring data integrity for real-time delivery.

2. Design Choices & Rationale

  • Adaptive Delay: We chose a fixed playout delay (194ms) to balance the trade-off between playback smoothness and end-to-end latency.
  • Efficient ARQ: The NACK-based recovery mechanism was chosen to maintain data continuity while keeping the bandwidth overhead below the 2.00x cap.
  • Scalability: Using std::map for the buffer allows for efficient insertion and lookups, which is critical for maintaining performance under heavy NACK traffic in high-loss environments like Profile B.

3. System-Level Performance

The system was stress-tested across varying network profiles to ensure robustness. The chosen delay of 194ms successfully resolves packet loss rates exceeding 10% in high-jitter environments while remaining within bandwidth constraints. This architecture effectively bridges the gap between unreliable network transport and the need for a stable, sequential stream for end-user playback.
